# ai-dev-workflow-manager

AI 开发工作流管理平台，第一版定位为个人 AI 开发流程管理与交付复盘工具。

本项目不替代 AI 编程工具，也不直接接入真实 AI 自动执行。它关注的是开发者如何把一次临时 AI 对话，沉淀成可配置、可复用、可复盘的工程化流程。

## 项目背景

AI 编程工具已经能帮助开发者生成代码、解释问题、修复缺陷，但实际开发中仍然存在几个常见问题：

- 任务复杂度和风险没有被提前识别，容易把高风险需求当成普通对话处理。
- 不同任务应该使用不同 workflow，但实际执行时往往依赖个人经验临时组织提示词。
- AI 输出后的阶段状态、测试清单、交付记录缺少统一沉淀。
- 求职、复盘、自媒体内容输出时，很难展示 AI 开发过程中的工程化方法。

## 项目定位

`ai-dev-workflow-manager` 对外可以作为 AI 开发工作流管理平台展示，但第一版 MVP 更准确的定位是个人 AI 开发流程管理与交付复盘工具。

它服务于 Codex、Claude Code 等 AI 编程工具的使用者，帮助开发者管理任务分级、风险标签、workflow 模板匹配、执行指令生成、阶段状态流转、测试清单和交付记录。

它解决的不是“AI 会不会写代码”的问题，而是“如何稳定、可控、可复盘地使用 AI 做开发”的问题。

第一版核心闭环：

```text
输入一个开发任务
↓
系统根据任务类型、复杂度、风险标签匹配 workflow
↓
生成阶段化执行 prompt
↓
人工把 Codex / Claude Code 的输出回填到阶段中
↓
系统生成测试清单、风险说明、交付总结
↓
导出 Markdown 复盘记录
```

## 核心问题

- 如何根据任务类型、复杂度和风险标签选择合适 workflow。
- 如何把 workflow 拆成分析、设计、实现、审查、验证、交付等阶段。
- 如何为每个阶段生成适合复制给 AI 编程工具的执行指令。
- 如何记录阶段输出、测试清单、交付总结和最终 Markdown。
- 如何明确哪些高风险任务不能跳过分析、风险评审和测试。

## 第一版 MVP 范围

第一版只做个人开发任务的流程管理、指令生成和交付记录：

- 创建和管理 AI 开发任务。
- 维护 workflow 模板、阶段模板和 prompt 模板。
- 根据任务类型、复杂度、风险标签匹配 workflow 模板。
- 推进任务阶段并记录阶段输出。
- 生成可复制给 Codex / Claude Code 的阶段指令。
- 生成测试清单、交付总结和 Markdown 导出内容。

## 核心功能

- 任务管理：记录任务标题、描述、类型、复杂度、风险标签和当前状态。
- 模板管理：维护 workflow 模板、阶段顺序、阶段目标和默认 prompt。
- 模板匹配：根据任务属性匹配推荐 workflow。
- 状态流转：约束任务状态和阶段状态，避免跳过关键步骤。
- 指令生成：生成阶段化、可复制的 AI 执行指令。
- 交付记录：沉淀测试清单、交付总结、风险说明和导出内容。

## 技术栈规划

- 后端：Spring Boot、MyBatis-Plus、MySQL、Knife4j。
- 前端：Vue 3、Element Plus、TypeScript。
- 数据库：MySQL。
- 文档与演示：Markdown、README、演示截图。

当前项目已进入 MVP 主链路开发阶段：

- 已创建 Spring Boot 后端骨架。
- 已创建 Vue 3 前端骨架。
- 后端已有 `task`、`template`、`matching`、`stage`、`prompt`、`delivery` 等 MVP 主链路基础模块。
- 前端已有 Vue Router 页面骨架、Axios API 封装和 mock 数据。
- 下一步重点是连接真实 MySQL 环境，完成前后端真实接口联调。

## 核心模块

- `task`：任务创建、查询、状态管理。
- `template`：workflow 模板、阶段模板、prompt 模板管理。
- `matching`：模板匹配规则。
- `stage`：任务阶段初始化、推进、输出记录。
- `prompt`：阶段指令、测试清单、交付总结生成。
- `delivery`：交付记录和 Markdown 导出。

## 数据模型概览

- `workflow_task`：AI 开发任务主表。
- `workflow_template`：workflow 模板主表。
- `workflow_template_stage`：模板阶段定义。
- `workflow_stage`：任务实际执行阶段。
- `prompt_template`：阶段指令模板。
- `delivery_record`：交付记录。
- `risk_tag`：可选风险标签字典。

## 状态流转概览

任务状态：

`DRAFT -> ANALYZING -> EXECUTING -> TESTING -> DELIVERED -> ARCHIVED`

异常或放弃时可进入 `CANCELED`。

阶段状态：

`PENDING -> RUNNING -> COMPLETED`

阶段也可以被标记为 `SKIPPED` 或 `FAILED`。高风险任务不能跳过分析和风险评审，未生成测试清单不能交付，已归档任务不能修改。

- 第 2 周：实现任务、模板、阶段核心接口。
- 第 3 周：实现模板匹配、状态机约束和指令生成。

## 暂不做内容

- 不接入真实 AI 自动执行。
- 不接入 Codex CLI / Claude CLI。
- 不做 RAG。
- 不做代码扫描。
- 不做模板市场。
- 不做复杂权限系统。

## 面试讲解价值

这个项目的展示价值不是“做了一个 AI 平台”，而是展示对 AI 辅助开发工程化的理解：不仅会使用 AI 生成代码，还能把任务分级、风险控制、workflow 模板、阶段交付、测试验证和复盘记录组合成一个可落地的闭环。

它也适合作为 GitHub 展示项目、自媒体内容素材和个人长期工具的起点。
