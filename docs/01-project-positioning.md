# 项目定位

## 为什么做这个项目

Codex、Claude Code 等 AI 编程工具已经能处理大量编码任务，但真实开发过程不仅是“让 AI 写代码”。一个可交付的开发任务通常还需要需求澄清、风险识别、方案设计、阶段执行、测试验证、交付总结和复盘记录。

如果这些过程只停留在一次性聊天记录里，后续很难复用，也很难向他人说明自己如何控制质量。因此，本项目希望先把个人 AI 开发过程中的流程管理、交付记录和复盘部分产品化。

## 定位分层

对外简历名称：

```text
AI 开发工作流管理平台
```

第一版 MVP 实际定位：

```text
个人 AI 开发流程管理与交付复盘工具
```

这样既保留项目的完整表达，也避免第一版在不接入 AI、不扫描代码、不自动执行、不做团队协作的情况下显得名大于实。

## 和 Codex / Claude Code 的关系

本项目不是 Codex / Claude Code 的替代品。

它更像是这些工具上层的个人流程管理与交付记录系统：

- Codex / Claude Code 负责具体分析、编码、解释和修改。
- 本项目负责管理任务、匹配 workflow、生成阶段指令、记录阶段输出和交付结果。

第一版不会接入 Codex CLI、Claude CLI，也不会自动调用模型。系统生成的指令由开发者复制到对应 AI 编程工具中执行。

## 第一版核心闭环

```text
输入一个开发任务
↓
系统根据任务类型、复杂度、风险标签匹配 workflow
↓
生成阶段化执行 prompt
↓
人工把 Codex / Claude Code 的输出回填到阶段中
↓
系统生成测试清单、风险说明、交付总结
↓
导出 Markdown 复盘记录
```

## 解决的问题边界

本项目解决的不是 AI 写代码能力问题，而是 AI 开发流程管理问题。

重点包括：

- 任务是否需要分级。
- 是否存在支付、权限、数据库、配置、文件等高风险标签。
- 应该使用哪类 workflow 模板。
- 哪些阶段必须完成，哪些阶段可以跳过。
- 交付前是否已经生成测试清单。
- 最终交付记录是否可复盘、可展示。

## 面试中的 30 秒介绍

`ai-dev-workflow-manager` 是一个面向 Codex、Claude Code 等 AI 编程工具的个人 AI 开发流程管理与交付复盘工具。它不替代 AI 写代码，而是管理 AI 开发任务的分级、风险标签、workflow 模板匹配、阶段状态、测试清单和交付记录，帮助开发者把临时 AI 对话沉淀成可复用、可复盘的工程化流程。

## 面试中的 1 分钟介绍

我做这个项目的原因是，AI 编程工具已经能提升编码效率，但很多人使用 AI 开发时缺少流程控制：高风险任务没有提前识别，prompt 靠临时发挥，测试和交付记录也不稳定。

这个项目把 AI 开发过程抽象成任务、workflow 模板、阶段、prompt 模板和交付记录。用户创建任务后，系统根据任务类型、复杂度和风险标签匹配 workflow，生成每个阶段可复制给 Codex 或 Claude Code 的执行指令。开发者把 AI 工具的输出回填到阶段记录中，系统再生成测试清单、风险说明、交付总结和 Markdown 复盘记录。

第一版不会接入真实 AI 自动执行，也不会做代码扫描或 RAG，重点是把流程管理闭环做清楚，适合求职展示、GitHub 项目展示、自媒体内容和个人长期工具沉淀。
