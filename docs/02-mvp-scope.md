# MVP 范围

## 第一版目标

第一版只做个人 AI 开发任务的流程管理、指令生成和交付记录。

系统不自动执行开发任务，不接入真实 AI，也不扫描代码仓库。用户在系统中创建任务、选择或匹配 workflow、推进阶段，并把系统生成的阶段指令复制到 Codex / Claude Code 中执行。

第一版要完成的核心闭环：

```text
输入一个开发任务
↓
系统根据任务类型、复杂度、风险标签匹配 workflow
↓
生成阶段化执行 prompt
↓
人工把 Codex / Claude Code 的输出回填到阶段中
↓
系统生成测试清单、风险说明、交付总结
↓
导出 Markdown 复盘记录
```

## P0 功能

- 任务创建：录入标题、描述、任务类型、复杂度、风险标签。
- 任务查询：查看任务列表、任务详情、当前状态和阶段进度。
- workflow 模板管理：维护模板名称、适用任务类型、复杂度、风险标签、优先级和版本。
- 阶段管理：根据模板生成任务阶段，支持阶段推进和输出记录。
- 指令生成：根据任务信息、阶段信息和 prompt 模板生成可复制指令。
- 测试清单生成：根据任务类型和风险标签生成基础测试清单。
- 交付记录：记录交付总结、测试结果、风险说明和 Markdown 导出内容。
- 状态约束：高风险任务不能跳过分析和风险评审，未生成测试清单不能交付。

## P1 功能

- 模板复制和版本升级。
- prompt 模板变量预览。
- 任务阶段输出的历史记录。
- Markdown 导出格式优化。
- 简单统计：任务数量、交付数量、不同风险标签分布。

## P2 功能

- 模板推荐说明：展示为什么命中某个模板。
- 多模板对比。
- 常用任务类型预设。
- 交付记录检索。
- 演示数据初始化脚本。

## 明确不做

- 不做自动执行。
- 不接入 Codex CLI / Claude CLI。
- 不接入真实 AI API。
- 不做代码扫描。
- 不做 RAG。
- 不做复杂权限系统。
- 不做模板市场。
- 不做多人协作审批流。
- 不生成完整 Spring Boot 项目或 Vue 项目。

## MVP 验收标准

- 能创建一个 AI 开发任务。
- 能根据任务属性匹配一个 workflow 模板。
- 能生成任务阶段并推进状态。
- 能记录阶段输出。
- 能生成阶段执行指令。
- 能回填 Codex / Claude Code 的阶段输出摘要。
- 能生成测试清单和交付总结。
- 能导出 Markdown 交付记录。
