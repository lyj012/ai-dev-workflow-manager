# Prompt 生成规则

## 生成目标

Prompt 生成的目标是把任务信息、workflow 阶段和历史阶段输出组合成一段可复制给 Codex / Claude Code 的执行指令。

第一版只生成文本，不调用真实 AI，不追踪外部 AI 执行状态。

## 输入信息

生成阶段 prompt 时建议使用以下信息：

- 任务标题。
- 任务描述。
- 任务类型。
- 复杂度。
- 风险标签。
- 当前 workflow 模板。
- 当前阶段名称和阶段目标。
- 当前阶段是否必需。
- 已完成阶段的输出摘要。
- 当前阶段的 prompt 模板。

## 基础结构

阶段 prompt 建议包含：

```text
你正在处理一个 AI 辅助开发任务。

任务信息：
- 标题：{{taskTitle}}
- 描述：{{taskDescription}}
- 类型：{{taskType}}
- 复杂度：{{complexity}}
- 风险标签：{{riskTags}}

当前阶段：
- 阶段：{{stageName}}
- 目标：{{stageGoal}}
- 必需：{{required}}

历史上下文：
{{previousStageOutputs}}

执行要求：
{{stageInstructions}}

输出要求：
请输出本阶段的结论、修改建议、风险点、验证建议和下一步动作。
```

## 高风险任务增强规则

当任务包含高风险标签时，生成 prompt 时必须附加：

- 明确要求识别风险点。
- 明确要求说明影响范围。
- 明确要求给出验证方式。
- 明确要求标注未验证范围。
- 涉及权限、支付、数据库、配置、文件处理时，要求避免只给出代码结论。

## 阶段差异

分析阶段重点输出：

- 需求目标。
- 影响范围。
- 风险标签。
- 关键约束。
- 是否需要进一步澄清。

设计阶段重点输出：

- 实现方案。
- 涉及模块。
- 接口或数据结构变化。
- 状态流转。
- 验证方案。

实现阶段重点输出：

- 修改范围。
- 关键代码变化。
- 兼容性说明。
- 异常处理。

审查阶段重点输出：

- 潜在缺陷。
- 状态一致性问题。
- 权限和边界问题。
- 测试缺口。

测试阶段重点输出：

- 测试清单。
- 已验证范围。
- 未验证范围。
- 剩余风险。

交付阶段重点输出：

- 交付摘要。
- 测试结果。
- 风险说明。
- 后续建议。
