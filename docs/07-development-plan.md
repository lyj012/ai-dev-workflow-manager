# 开发计划

## 第 1 周：文档、表结构、后端骨架

目标：

- 完成 README 和核心设计文档。
- 完成 MySQL 建表草案。
- 明确后端模块规划。
- 明确前端页面规划。

验收标准：

- README 能说明项目定位、MVP 范围、技术栈和暂不做内容。
- `docs/` 下包含项目定位、MVP、数据库、API、状态机、模板匹配和开发计划。
- `sql/schema.sql` 能表达核心表结构。
- `backend/README.md` 和 `frontend/README.md` 说明后续建设方向。

## 第 2 周：任务、模板、阶段核心接口

目标：

- 创建 Spring Boot 基础项目。
- 实现任务创建、任务查询接口。
- 实现 workflow 模板和模板阶段管理接口。
- 实现任务阶段初始化和阶段推进接口。

验收标准：

- 能通过接口创建任务。
- 能维护 workflow 模板和阶段定义。
- 能根据模板初始化任务阶段。
- 能推进阶段状态并记录阶段输出。

## 第 3 周：模板匹配、状态机、指令生成

目标：

- 实现基于 `task_type + complexity + risk_tags` 的模板匹配。
- 实现任务状态和阶段状态流转校验。
- 实现阶段指令生成。
- 补充高风险任务约束。

验收标准：

- 普通任务能匹配基础 workflow。
- 高风险任务优先匹配包含分析、风险评审和测试要求的 workflow。
- 不允许跳过高风险任务的关键阶段。
- 能生成可复制给 Codex / Claude Code 的阶段指令。

## 第 4 周：测试清单、交付总结、Markdown 导出

目标：

- 实现测试清单生成。
- 实现交付总结生成。
- 实现 Markdown 导出。
- 完善 README、演示截图和示例数据。

验收标准：

- 未生成测试清单的任务不能交付。
- 交付记录包含任务信息、阶段输出、测试清单、风险说明和交付总结。
- Markdown 导出内容可用于 GitHub 展示和个人复盘。
- README 能支持求职讲解和项目展示。

## 后续方向

第一版完成后，再考虑是否增加：

- 更细的模板变量系统。
- 任务统计面板。
- 多模板对比。
- Codex / Claude Code CLI 接入。
- AI API 接入。
- 代码扫描或 RAG。

这些内容不属于第一版 MVP。
