# MVP 演示流程

## 目标

本文档用于稳定化阶段的真实联调、演示录屏和面试讲解。

演示重点不是自动执行 AI 开发任务，而是展示系统如何把一次 AI 辅助开发过程管理成可复盘的 workflow。

## 演示前提

- 后端服务可访问。
- 前端使用真实接口模式，不启用 `VITE_USE_MOCK=true`。
- MySQL 已创建表结构。
- 内置 workflow 模板已初始化。

## 推荐演示任务

标题：

```text
补齐任务详情页阶段状态联动
```

描述：

```text
任务详情页在阶段开始、完成、跳过或失败后，需要同步更新当前阶段列表和选中阶段展示，避免前端显示旧状态。
```

任务类型：

```text
bugfix
```

复杂度：

```text
MEDIUM
```

风险标签：

```text
无
```

## 主链路步骤

### 1. 创建任务

进入任务创建页，填写任务标题、描述、任务类型、复杂度和风险标签。

预期结果：

- 创建成功后进入任务详情页。
- 任务初始状态为 `DRAFT`。
- 尚未绑定 workflow 模板。

### 2. 匹配 workflow

在任务详情页点击“匹配 workflow”。

预期结果：

- 系统返回匹配结果。
- 普通中等复杂度任务优先匹配标准开发 workflow。
- 匹配结果展示模板名称、分数、自动绑定状态和匹配原因。
- 自动绑定成功时，任务详情展示命中的模板名称。

### 3. 初始化阶段

点击“初始化阶段”。

预期结果：

- 根据命中的 workflow 模板生成实际任务阶段。
- 阶段初始状态为 `PENDING`。
- 页面展示阶段顺序、阶段名称、状态和操作入口。

### 4. 推进阶段

选择第一个阶段，执行“开始”。

预期结果：

- 阶段状态从 `PENDING` 变为 `RUNNING`。
- 页面阶段列表同步更新。

继续执行“完成”。

预期结果：

- 阶段状态从 `RUNNING` 变为 `COMPLETED`。
- 页面阶段列表同步更新。

### 5. 生成阶段 Prompt

选择一个阶段，点击“生成 Prompt”。

预期结果：

- Prompt 包含任务标题、描述、类型、复杂度、风险标签。
- Prompt 包含当前阶段目标。
- Prompt 包含历史阶段输出上下文。
- Prompt 可复制给 Codex / Claude Code。

### 6. 回填阶段输出

点击“编辑输出”，填写：

- 输出摘要。
- 风险点。
- 后续动作。
- 未验证范围。

保存后预期结果：

- 当前阶段输出区展示最新内容。
- 阶段列表中的输出摘要同步更新。

### 7. 生成测试清单

点击“生成测试清单”。

预期结果：

- 系统根据任务类型、复杂度和风险标签生成测试清单。
- 任务标记为已生成测试清单。

### 8. 生成交付总结

在必需阶段满足交付约束后，点击“生成交付总结”。

预期结果：

- 系统校验测试清单已生成。
- 系统校验必需阶段已完成。
- 系统生成交付总结、风险说明和 Markdown 内容。

### 9. 预览 Markdown

进入交付预览页。

预期结果：

- 页面展示任务信息。
- 页面展示 workflow 阶段记录。
- 页面展示测试清单、测试结果、风险说明、未验证范围和交付总结。
- Markdown 内容可用于个人复盘、GitHub 展示或面试讲解。

## 高风险任务演示补充

如需展示高风险约束，可以创建包含 `database`、`permission` 或 `auth` 风险标签的复杂任务。

预期结果：

- 匹配优先命中高风险 workflow。
- `analysis` 和 `risk_review` 阶段不能跳过。
- 未生成测试清单时不能生成最终交付总结。
- 交付记录必须包含风险说明和未验证范围。

## 不演示内容

第一版不演示以下能力：

- 真实 AI API 调用。
- Codex CLI / Claude CLI 自动执行。
- RAG。
- 代码扫描。
- 权限系统。
- 模板市场。
- 多人协作审批。
- 复杂统计大屏。
