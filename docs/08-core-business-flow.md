# 核心业务流程

## 流程目标

第一版目标不是自动完成开发任务，而是帮助开发者把一次 AI 辅助开发过程记录成可复用、可复盘、可展示的交付链路。

## 主流程

```text
创建任务
↓
填写任务类型、复杂度、风险标签
↓
匹配 workflow 模板
↓
初始化任务阶段
↓
生成当前阶段执行 prompt
↓
开发者复制 prompt 到 Codex / Claude Code
↓
开发者回填 AI 输出、关键结论、风险点和后续动作
↓
推进下一个阶段
↓
生成测试清单
↓
记录测试结果和未验证范围
↓
生成交付总结
↓
导出 Markdown 复盘记录
```

## 关键业务对象

- 任务：一次 AI 辅助开发工作的主记录。
- workflow 模板：不同任务类型、复杂度和风险标签对应的流程模板。
- 阶段：一次任务中的分析、设计、实现、审查、测试、交付等步骤。
- prompt 模板：用于生成可复制给 AI 编程工具的阶段指令。
- 阶段输出：开发者从 Codex / Claude Code 获得结果后回填的摘要。
- 交付记录：测试清单、风险说明、交付总结和 Markdown 导出内容。

## 高风险流程约束

当任务包含支付、权限、数据库、配置、文件处理、登录认证、下载权限、金额计算、回调处理等高风险标签时：

- 必须完成需求分析。
- 必须完成风险识别或风险评审。
- 不能跳过必需阶段。
- 交付前必须生成测试清单。
- 交付总结必须包含风险说明和未验证范围。

## 第一版不处理的流程

- 不自动调用 AI 模型。
- 不自动执行 Codex / Claude Code。
- 不扫描代码仓库。
- 不做团队审批。
- 不做多人权限。
- 不做模板市场。
